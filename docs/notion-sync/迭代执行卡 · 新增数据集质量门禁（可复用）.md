# 迭代执行卡 · 新增数据集质量门禁（可复用）

> 🎯 ****用途：** 每次新增一个数据集到横评体系时，按本卡 6 步流水线执行。全程约 10-20 分钟（取决于数据集大小和网络速度）。**
## 前置条件
- [x] ECS 上已有 `~/work/claude-code-robotics-playbook` 仓库（git clone 过）
- [x] ECS 上已安装 Python 依赖（`pip install -r requirements.txt`）
- [x] ECS 上 HF 镜像已配置（`HF_ENDPOINT=https://hf-mirror.com`）
- [x] Mac 与 ECS 之间 SSH 免密已配置（`ssh ecs-host hostname` 可直接返回）
- [x] Mac 本地仓库与 GitHub 同步正常
---
## 6 步流水线
### Step 1 ⬇️ ECS 下载新数据集（2-15 min）
```bash
# 替换 <org/dataset_name> 和 <local_dir_name>
ssh ecs-host 'source ~/.bashrc && \
  hf download --repo-type dataset <org/dataset_name> \
    --local-dir ~/datasets/<local_dir_name>'

# 验证下载完整性
ssh ecs-host 'du -sh ~/datasets/<local_dir_name> && \
  ls ~/datasets/<local_dir_name>/meta/info.json'
```
> 💡 ****数据集名去哪找？** [huggingface.co/lerobot](http://huggingface.co/lerobot) 搜关键词。优先选 episode ≥ 20、最近半年更新、stars > 5 的数据集。社区个人上传的数据集质量问题概率最高，适合找「问题素材」。**
### Step 2 🔄 Mac 推最新代码到 GitHub（1 min）
```bash
# 在 Mac 本地仓库
cd ~/work/claude-code-robotics-playbook
git add -A && git commit -m "feat: 门禁代码更新" && git push
```
> ⚠️ ****如果代码没有改动可以跳过此步。** 但如果你修改了 checks/ 或 run_[gate.py](http://gate.py/)，必须先 push 再让 ECS pull。**
### Step 3 🏃 ECS 拉代码 + 跑门禁（2-10 min）
```bash
# 一条命令搞定：拉代码 → 跑门禁 → 提交报告 → 推送
ssh ecs-host 'cd ~/work/claude-code-robotics-playbook && git pull && \
  python demos/data-quality-gate/scripts/run_gate.py \
    --dataset ~/datasets/<local_dir_name> \
    --output demos/data-quality-gate/reports/<local_dir_name> \
    --profile strict && \
  git add demos/data-quality-gate/reports/<local_dir_name>/ && \
  git commit -m "data: <local_dir_name> 门禁报告" && git push'
```
> 💡 ****如果门禁脚本在 ECS 上报依赖缺失：** 先跑 `ssh ecs-host 'cd ~/work/claude-code-robotics-playbook && pip install -r requirements.txt'`**
### Step 4 📥 Mac 拉报告回来（30 sec）
```bash
cd ~/work/claude-code-robotics-playbook
git pull
```
### Step 5 📊 查看报告（1 min）
```bash
# 查看 Markdown 报告
cat demos/data-quality-gate/reports/<local_dir_name>/report.md

# 查看 JSON 报告（机器可读）
cat demos/data-quality-gate/reports/<local_dir_name>/report.json | python -m json.tool
```
### Step 6 📝 更新横评总表（可选，写文章时用）
如果需要加入横评对比，让 CC CLI 读取所有 reports/ 子目录，重新生成横评总表：
```plain text
请读取 demos/data-quality-gate/reports/ 下所有数据集的报告，输出更新后的横评总表：

| 数据集 | exit_code | 帧率CV | 缺帧率 | 多视角偏移 | 标签缺失率 | 异常帧率 | 总评 |

新增的数据集用 🆕 标记。如果有 exit_code != 0 的，深挖具体问题 episode。
```

## 快速复制模板
> 📋 ****一键执行版：** 把下面整段命令中的 `<org/dataset_name>` 和 `<local_dir_name>` 替换后，直接贴到终端执行。**
```bash
# === 替换这两个变量 ===
DS_HF="<org/dataset_name>"          # 例: lerobot/aloha_static_coffee
DS_LOCAL="<local_dir_name>"          # 例: aloha_static_coffee

# === 以下不用改 ===
# Step 1: ECS 下载
ssh ecs-host "source ~/.bashrc && hf download --repo-type dataset $DS_HF --local-dir ~/datasets/$DS_LOCAL"

# Step 2: Mac push 代码（如有改动）
cd ~/work/claude-code-robotics-playbook && git add -A && git diff --cached --quiet || git commit -m 'feat: 门禁更新' && git push

# Step 3: ECS 跑门禁 + 提交报告
ssh ecs-host "cd ~/work/claude-code-robotics-playbook && git pull && \
  python demos/data-quality-gate/scripts/run_gate.py \
    --dataset ~/datasets/$DS_LOCAL \
    --output demos/data-quality-gate/reports/$DS_LOCAL \
    --profile strict && \
  git add demos/data-quality-gate/reports/$DS_LOCAL/ && \
  git commit -m \"data: $DS_LOCAL 门禁报告\" && git push"

# Step 4: Mac 拉报告
git pull

# Step 5: 查看报告
cat demos/data-quality-gate/reports/$DS_LOCAL/report.md
```

```plain text
执行步骤：

2. Mac 上 git push 最新代码
3. ssh ecs-host 拉代码 + 跑 strict 门禁 + git commit 报告 + git push

```

## 常见问题
| 问题 | 原因 | 解决 |
| --- | --- | --- |
| ECS 磁盘满 | 数据集太大 | `ssh ecs-host 'df -h'` 检查；删除不需要的旧数据集或挂数据盘 |
| hf download 报错 404 | 数据集名拼错或已下架 | 去 [huggingface.co](http://huggingface.co/) 确认数据集存在且名称正确 |
| git push 冲突 | Mac 和 ECS 同时改了文件 | `git pull --rebase` 解决冲突后重新 push |
| 门禁脚本报 ImportError | ECS 缺依赖 | `ssh ecs-host 'pip install -r ~/work/claude-code-robotics-playbook/requirements.txt'` |
| 数据集不是 LeRobot 格式 | 非标准格式 | 检查是否有 `meta/info.json`；如果是 rosbag/其他格式需要先转换 |
